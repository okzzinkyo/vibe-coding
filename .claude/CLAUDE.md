# 브리프 규칙
- 한국어로 대화한다
- 코드 수정 전에 먼저 설명한다
- 파일 변경 후 테스트 실행한다
- 요청이 불완전하면 작업 전에 반드시 질문한다
- 모호한 부분은 추측하지 말고 확인한다
- 질문은 한 번에 몰아서 (여러 번 왔다갔다 금지)

# 파일 관리 규칙
- 프로젝트는 루트에 바로 폴더 생성 (예: ./lab-inventory/)
- 단순 결과물(보고서, 문서 등)은 ./결과물/ 폴더에 저장
- 폴더 없으면 자동 생성 (mkdir -p)
- 파일명은 날짜_내용 형식으로 (예: 20260411_보고서.md)

# 의존성 & 빌드 규칙
- 라이브러리 설치 전 버전 확인 (npm info <패키지> version), 설치 후 package.json 기준으로 공식 설정을 따른다
- CLI 명령어 실행 전 해당 바이너리가 node_modules/.bin에 존재하는지 확인한다

# UI 구현 규칙
- 방향이 크게 달라질 수 있는 UI(레이아웃, 네비게이션 방식 등)는 코드 작성 전 선택지와 장단점을 제시하고 확인 후 구현한다

# git 커밋 규칙
- 커밋 전 git status로 파일 목록 확인, 불필요 파일(.env, node_modules 등)은 .gitignore 처리 후 커밋한다
- 새 프로젝트 폴더 생성 시 npm install 전에 .gitignore부터 작성한다

# 교훈
- 파괴적 동작(삭제, 초기화 등)은 커스텀 확인 다이얼로그를 사용한다. 브라우저 기본 confirm() 금지.
- 실행으로 검증 가능한 값(쿼리 결과, 계산값 등)은 AI 추론에 의존하지 않고 코드로 직접 실행해서 얻는다.
