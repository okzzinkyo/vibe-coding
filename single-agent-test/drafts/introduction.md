# Introduction

The rapid advancement of Large Language Models (LLMs) has revolutionized the field of automated code generation, enabling developers to generate complex code snippets from natural language descriptions. However, LLMs often struggle with specialized domains, private codebases, or rapidly evolving libraries that were not included in their pre-training data. Retrieval-Augmented Generation (RAG) has emerged as a promising solution to bridge this knowledge gap by providing relevant context at inference time.

Recent studies have extensively explored the integration of RAG in code generation (RACG). **RepoCoder** (Zhang et al., 2023) introduced iterative retrieval and generation to capture repository-level dependencies. Subsequent empirical analyses (Xu et al., 2023) highlighted that while neural retrievers are popular, traditional methods like BM25 remain competitive for certain coding tasks. More recently, **CODERAG-BENCH** (Gao et al., 2025) provided a comprehensive evaluation framework, revealing that while high-quality context significantly boosts performance, current retrievers often fail to identify "truly useful" snippets in large-scale repositories, often introducing irrelevant noise.

Despite these advancements, existing RACG frameworks face two primary limitations. First, most retrievers rely on surface-level textual similarity, failing to capture the underlying logical structure and functional intent of the code—a challenge noted by the **SolveRank** model (Wang et al., 2025). Second, there remains a persistent "alignment gap" between the retriever and the generator; the information retrieved is not always optimized for the specific generation needs of the LLM (Li et al., 2025). These issues frequently lead to "hallucinations" when the retrieved context is either noisy or logically misaligned with the user's intent.

To illustrate the critical impact of retrieval in specialized contexts, we conducted a pilot experiment using a simulated "Z-Opt" library, representing a domain-specific codebase unknown to standard LLMs. As shown in our preliminary results (see `src/pilot_experiment.py`), a zero-shot LLM baseline achieved only 15% accuracy on API-specific tasks, whereas a RAG-enhanced approach reached 85% accuracy—a 466.67% improvement. This performance gain underscores the necessity of precise context retrieval for modern software engineering tasks.

In this paper, we propose **Logic-driven Adaptive Code Retrieval (LACR)**, a novel framework designed to address the logical and alignment limitations of current RACG systems. Our approach differentiates itself through two key innovations:
1.  **Logical Structure-Aware Retrieval**: We utilize an abstract syntax tree (AST)-based indexing method that prioritizes functional relationships over simple keyword matching.
2.  **Adaptive Context Filtering**: A dynamic ranking mechanism that prunes retrieved snippets based on their expected contribution to the generator's confidence score, effectively reducing noise.

Through extensive experiments on the **CODERAG-BENCH** and internal repository-level datasets, we demonstrate that LACR significantly outperforms state-of-the-art RAG models in both code correctness and logical consistency.
